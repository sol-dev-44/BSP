<objective>
Store `slot_type` and `per_person_rate` when creating a booking, then use those stored values on the success page to show a correct itemized receipt. Also add the missing `combo_package` line item to the success page.

Purpose: The success page currently hardcodes $119/person for all slot types (BUG B), so an Early Bird booking shows "$238.00 for 2 people" when the total paid was $198.00. This makes the receipt internally inconsistent and erodes trust. Storing the authoritative values at booking creation time gives the success page an immutable receipt to display.

Output: bookings API route stores two new fields on insert; success page reads them and renders correct line items for all slot types and add-ons including combo_package.
</objective>
