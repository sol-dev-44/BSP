3. Select and copy the ENTIRE §10b block. It contains four logical SQL statements:
   - `ALTER TABLE bsp_discount_codes ADD COLUMN IF NOT EXISTS ...` (three columns)
   - Three `COMMENT ON COLUMN ...` statements
   - `CREATE OR REPLACE FUNCTION bsp_increment_discount_redemption(p_code_name TEXT) ...`
   - `INSERT INTO bsp_discount_codes ... ON CONFLICT (code_name) DO UPDATE ...`

4. Paste into the SQL Editor and click "Run".

5. Expected result: all statements succeed. Common success indicators:
   - "Success. No rows returned" for ALTER, COMMENT, CREATE FUNCTION.
   - "Success. 1 row affected" for the INSERT (or "Success. No rows returned" if 30OFF already existed — ON CONFLICT path).

6. Verify the columns exist with this query in a new SQL Editor tab:
   ```sql
   SELECT column_name, data_type, is_nullable, column_default
   FROM information_schema.columns
   WHERE table_name = 'bsp_discount_codes'
     AND column_name IN ('max_redemptions', 'times_redeemed', 'excludes_early_bird');
   ```
   Expect three rows returned. Each `is_nullable` = 'NO'. `max_redemptions` and `times_redeemed` default `0`; `excludes_early_bird` defaults `false`.

7. Verify the RPC exists:
   ```sql
   SELECT proname, prorettype::regtype, proargnames
   FROM pg_proc
   WHERE proname = 'bsp_increment_discount_redemption';
   ```
   Expect one row.

8. Verify the seed exists:
   ```sql
   SELECT code_name, amount, max_redemptions, times_redeemed, excludes_early_bird, is_active
   FROM bsp_discount_codes
   WHERE code_name = '30OFF';
   ```
   Expect exactly: `30OFF | 30 | 6 | 0 | true | true` (or `times_redeemed` may be higher if you've already test-booked — that's fine per D-15's ON CONFLICT preserving counter).

9. Spot-test the RPC manually (optional but recommended):
   ```sql
   SELECT * FROM bsp_increment_discount_redemption('30OFF');
   ```
   Expect a single row with `times_redeemed = previous + 1` and `at_cap = false` (assuming previous < 6). Repeat until at cap, then expect `at_cap = true` and `times_redeemed = NULL`. **Important: each test increment counts against the 30OFF cap of 6 — reset via `UPDATE bsp_discount_codes SET times_redeemed = 0 WHERE code_name = '30OFF';` if you exhaust it during testing.**
